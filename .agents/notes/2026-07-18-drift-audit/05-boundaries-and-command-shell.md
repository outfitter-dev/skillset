# 05 — Operations misfiled in the CLI + no shared command shell

Intent baseline: core returns facts/plans, app renders them
(`docs/adrs/drafts/20260612-core-library-boundary.md`,
`docs/package-ownership.md`). Verified clean first: no app-level re-export
shims (guard enforces), prompt-adapter boundary holds (zero `@clack` outside
`prompt-adapter.ts`), zero TTY/stdout leakage into `packages/` (sole
`process.stdout` under packages is toolkit's own CLI). The gaps are altitude
misfilings and duplication of the command scaffold. Sequence matters here — see
batch shape.

### 05.1 — No shared command shell: 13 `*-cli.ts` files re-implement parse→execute→(json|render)→exit (confirmed, HIGH payoff / small risk)
- 37 `printCliJsonData` call sites; `process.exitCode = 1` hand-set in 11 files
  (sometimes redundantly — `printCliJsonData` already sets it); 259 raw
  `console.log/error` calls concentrated in `*-cli.ts` (inspect 41, change 33,
  init 25…). Anchors: `build-cli.ts:20-101`, `check-cli.ts:37-104`,
  `init-cli.ts:40-189`.
- **Fix:** one `runCommand({command, jsonOutput}, exec, renderHuman)` shell
  owning the json fork + exit mapping; command files shrink to op + renderer.
  Mechanical, migrate one file at a time.

### 05.2 — `test-runner.ts` (1622 LOC) is a headless core operation living in the app (confirmed, HIGH payoff / medium risk)
- Exports structured report operations; reaches into **eight**
  `@skillset/core/internal/*` modules — the biggest driver of the app's 55
  internal imports. Nothing argv/prompt/stdout in it; `test-cli.ts` is already
  the thin client.
- **Fix:** move to core as an operation (per ownership doc: core owns
  "verify/conformance operations… operation results").

### 05.3 — `ciSkillset` orchestration (560 LOC `ci.ts`) belongs in core (confirmed, HIGH payoff / medium risk)
- Pure orchestration returning `CiReport`; `check-cli.ts` wraps it in ~240
  lines of presentation. App-side placement blocks non-CLI consumers, which the
  boundary ADR explicitly wants enabled. Markdown/terminal renderers stay
  app-side.
- Interacts with note 06.1 (terminal recovery hint lives in this seam).

### 05.4 — `provider-maintenance.ts` (667 LOC): registry codegen owned by the app, reachable only via scripts (confirmed, MEDIUM)
- Regenerates `packages/registry/src/schema-snapshots.ts` from
  `apps/skillset/src/provider-maintenance.ts`, called only by
  `scripts/provider-maintenance.ts` glue (`providers:*` npm scripts). A
  package's source artifact generated by a different package at the wrong
  altitude.
- **Fix:** relocate next to what it maintains (registry, or a core maintenance
  module); script/subcommand stays a thin caller.

### 05.5 — Scaffold-report rendering copy-pasted across init/create/new/import/adopt (confirmed, MEDIUM payoff / small risk)
- `init-cli.ts:191-306`, `source-cli.ts:250-293` + friends each hand-roll the
  "+/= file, next: …, rerun with --yes" report. The interactive half is already
  shared (`interactive-session.ts`); the report half isn't.
- **Fix:** shared `renderScaffoldReport` + one write-confirmation helper; rides
  on 05.1.

### 05.6 — Oversized files with real seams (confirmed, per-file)
- Required SET-337 scope: extract the independently testable `render.ts` seams
  for marketplace (:289-509), plugin-manifest (:574-708), rules (:1752-1975),
  and adaptive/native hooks (:2110+).
- Future candidates, not a file-count quota or implicit SET-337 scope:
  `resolver.ts` 1489 (island vs plugin/skill discovery vs validation);
  `feature-registry.ts` 1376 (push provider evidence registry-ward);
  `schema/validate.ts` 1306 (split per entity); app: `change-status.ts` 1254
  (computation is a core candidate), `adopt.ts` 1111 / `import.ts` 948 (shared
  machinery, see 05.5), `config.ts` 1076, `build.ts` 905.
- The ownership rule remains: split on ownership and testability, not size.

### 05.7 — `inspect-args.ts` (826 LOC) parses four commands; recurring `--json/--yes/--root` trio re-derived per parser (confirmed, LOW-MEDIUM)
- Don't over-consolidate parsers (flags are real surface). Do split
  inspect-args per subcommand and lift the shared trio into `cli-arg-reader`.

### 05.8 — After 05.2/05.3: promote stable internals out of `/internal` (proposal)
- Post-moves, remaining hot internal subpaths (`types`, `path`) become
  candidates for real `@skillset/core` root exports, per the ADR's
  "acceptable-but-temporary" framing of `/internal`.
- Minor normalization: `hooks-cli.ts:45,72` writes ad-hoc strings instead of
  going through a renderer — fold into 05.1 migration.

## Batch shape (sequence)
05.1 shell → 05.5 report renderer → 05.2 + 05.3 in parallel (boundary moves)
→ 05.4 → 05.6 required render.ts seams → 05.7/05.8 cleanup.
The first two are low-risk mechanical; the moves are where the architectural
payoff is.
